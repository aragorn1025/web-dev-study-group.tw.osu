# Topics
- Request and Respond
    - Client will submit a request message to a server
    - Server will return response message back to the client
    - HTTP is the rule of sending the message
- HTTP
    - contains request line, request headers and body
    - Request line
        - HTTP methods(GET/POST/PUT/DELETE)
        - request target(ex. URL)
        - HTTP version
    - Request header
        - Providing information about the request or the desired response
        - consist of a number of key-value content
    - body 
     - the actual content that we want to deliver
     - The formats of HTTP body
      - JSON (The most common)
      - HTML form (Application form)
      - Form Data (Uploading images and files)
      - text
    - Headers is the part that examines the format of body.
- Backend
    - accept the request and ask for the data from database
    - the languages that use in backend
        - Javascipt
            - ex. Node.js
        - python
            - ex. Flask, django, FastAPI
        - Go
            - gin
- Database
    - the place storing data
    - the place create, read, update, delete data
    - the type of database 
     - MangoDB (NoSQL)
     - Redis (NoSQL)
     - MySQL (SQL)
     - PostgreSQL (SQL)
- We will use SQL or ORM to make some operations between backend and database
- API
    - a method of communication between frontend and backend
    - the content of API needs to include
     - which URL we will use
     - which method? GET or POST?
     - what will put in the body part?
     - which type of data will database return ?
